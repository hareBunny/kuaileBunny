# 服务器部署指南

## 服务器环境要求

- **操作系统**: Linux (Ubuntu 20.04+ / CentOS 7+)
- **Node.js**: 18.x 或更高
- **内存**: 至少 1GB
- **磁盘**: 至少 10GB

---

## 一、安装基础环境

### 1. 安装 Node.js

```bash
# Ubuntu/Debian
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt-get install -y nodejs

# CentOS/RHEL
curl -fsSL https://rpm.nodesource.com/setup_18.x | sudo bash -
sudo yum install -y nodejs

# 验证安装
node -v
npm -v
```

### 2. 安装 PM2

```bash
sudo npm install -g pm2
pm2 -v
```

### 3. 安装 Wrangler

```bash
sudo npm install -g wrangler
wrangler -v
```

## 二、部署后端（Cloudflare Workers）

### 1. 上传代码

```bash
# 使用 Git
git clone your-repo-url
cd backend-cloudflare

# 或使用 SCP
scp -r backend-cloudflare user@server:/path/to/project/
```

### 2. 安装依赖

```bash
cd backend-cloudflare
npm install
```

### 3. 配置环境变量

创建 `.dev.vars` 文件：

```bash
cat > .dev.vars << EOF
JWT_SECRET=your-super-secret-jwt-key-min-32-characters
CRON_SECRET=your-cron-secret-key
LOTTERY_API_ID=your-api-id
LOTTERY_API_KEY=your-api-key
EOF
```

### 4. 初始化数据库

```bash
# 登录 Cloudflare
wrangler login

# 初始化数据库
wrangler d1 execute kuaile8 --file=./schema.sql --remote
```

### 5. 部署到 Cloudflare

```bash
wrangler deploy
```

## 三、部署爬虫定时任务

### 1. 进入爬虫目录

```bash
cd backend-cloudflare/crawler
```

### 2. 安装依赖

```bash
npm install
```

### 3. 测试爬虫

```bash
npm run crawl
```

### 4. 启动定时任务

#### 方式A: 使用 PM2（推荐）

```bash
# 启动定时任务
pm2 start ecosystem.config.cjs

# 查看状态
pm2 status

# 查看日志
pm2 logs kuaile8-crawler

# 查看实时日志
pm2 logs kuaile8-crawler --lines 100

# 停止
pm2 stop kuaile8-crawler

# 重启
pm2 restart kuaile8-crawler

# 删除
pm2 delete kuaile8-crawler
```

#### 方式B: 使用系统 Cron

编辑 crontab：

```bash
crontab -e
```

添加定时任务（每天晚上12点）：

```bash
0 0 * * * cd /path/to/backend-cloudflare/crawler && node official-crawler.js >> crawler.log 2>&1
```

保存并退出。

查看定时任务：

```bash
crontab -l
```

## 五、部署 H5 前端

### 1. 构建前端

```bash
cd h5
npm install
npm run build
```

### 2. 安装 Nginx

```bash
# Ubuntu/Debian
sudo apt-get install nginx

# CentOS/RHEL
sudo yum install nginx
```

### 3. 配置 Nginx

创建配置文件：

```bash
sudo nano /etc/nginx/sites-available/kuaile8
```

添加配置：

```nginx
server {
    listen 80;
    server_name your-domain.com;
    root /path/to/h5/dist;
    index index.html;

    # Gzip 压缩
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml application/xml+rss text/javascript;

    # SPA 路由支持
    location / {
        try_files $uri $uri/ /index.html;
    }

    # API 代理到 Cloudflare Workers
    location /api {
        proxy_pass https://your-worker.workers.dev;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # 静态资源缓存
    location ~* \.(jpg|jpeg|png|gif|ico|css|js|svg|woff|woff2|ttf|eot)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
}
```

启用配置：

```bash
sudo ln -s /etc/nginx/sites-available/kuaile8 /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx
```

### 4. 配置 HTTPS（可选但推荐）

使用 Let's Encrypt：

```bash
# 安装 Certbot
sudo apt-get install certbot python3-certbot-nginx

# 获取证书
sudo certbot --nginx -d your-domain.com

# 自动续期
sudo certbot renew --dry-run
```

## 六、监控与维护

### 1. 查看爬虫日志

```bash
# PM2 日志
pm2 logs kuaile8-crawler

# 文件日志
tail -f /path/to/crawler/crawler.log

# 查看最近100行
tail -n 100 /path/to/crawler/crawler.log
```

### 2. 查看数据库

```bash
cd backend-cloudflare

# 查看最新数据
wrangler d1 execute kuaile8 --command="SELECT * FROM lottery_results ORDER BY draw_date DESC LIMIT 10" --remote

# 查看今天的数据
wrangler d1 execute kuaile8 --command="SELECT COUNT(*) FROM lottery_results WHERE draw_no LIKE '$(date +%Y%m%d)%'" --remote
```

### 3. 监控脚本

创建 `monitor.sh`：

```bash
#!/bin/bash

# 检查 PM2 进程
if ! pm2 list | grep -q "kuaile8-crawler"; then
    echo "警告: 爬虫进程未运行"
    pm2 start ecosystem.config.cjs
fi

# 检查今天是否有数据
TODAY=$(date +%Y%m%d)
COUNT=$(wrangler d1 execute kuaile8 --command="SELECT COUNT(*) FROM lottery_results WHERE draw_no LIKE '${TODAY}%'" --remote | grep -oP '\d+')

if [ "$COUNT" -eq 0 ]; then
    echo "警告: 今天还没有数据"
    # 可以发送告警通知
fi
```

添加到 crontab（每小时检查一次）：

```bash
0 * * * * /path/to/monitor.sh >> /path/to/monitor.log 2>&1
```

### 4. 备份数据库

```bash
#!/bin/bash

# 导出数据
wrangler d1 export kuaile8 --output=backup-$(date +%Y%m%d).sql --remote

# 压缩
gzip backup-$(date +%Y%m%d).sql

# 删除7天前的备份
find . -name "backup-*.sql.gz" -mtime +7 -delete
```

添加到 crontab（每天凌晨2点备份）：

```bash
0 2 * * * /path/to/backup.sh >> /path/to/backup.log 2>&1
```

## 七、性能优化

### 1. Nginx 优化

```nginx
# 增加工作进程
worker_processes auto;

# 增加连接数
events {
    worker_connections 2048;
}

# 启用 HTTP/2
listen 443 ssl http2;
```

### 2. PM2 优化

```javascript
// ecosystem.config.cjs
module.exports = {
  apps: [{
    name: 'kuaile8-crawler',
    script: 'cron-config.js',
    instances: 1,
    max_memory_restart: '200M',
    error_file: './logs/error.log',
    out_file: './logs/out.log',
    log_date_format: 'YYYY-MM-DD HH:mm:ss'
  }]
}
```

### 3. 数据库优化

已经在 schema.sql 中创建了索引，无需额外优化。

## 八、安全配置

### 1. 防火墙

```bash
# Ubuntu
sudo ufw allow 22
sudo ufw allow 80
sudo ufw allow 443
sudo ufw enable

# CentOS
sudo firewall-cmd --permanent --add-service=ssh
sudo firewall-cmd --permanent --add-service=http
sudo firewall-cmd --permanent --add-service=https
sudo firewall-cmd --reload
```

### 2. 限制 SSH 访问

编辑 `/etc/ssh/sshd_config`：

```bash
PermitRootLogin no
PasswordAuthentication no
```

重启 SSH：

```bash
sudo systemctl restart sshd
```

### 3. 定期更新

```bash
# Ubuntu
sudo apt-get update && sudo apt-get upgrade -y

# CentOS
sudo yum update -y
```

## 九、故障排查

### 问题1: 爬虫不执行

**检查**:
```bash
pm2 status
pm2 logs kuaile8-crawler
```

**解决**:
```bash
pm2 restart kuaile8-crawler
```

### 问题2: 数据库连接失败

**检查**:
```bash
wrangler whoami
wrangler d1 list
```

**解决**:
```bash
wrangler login
```

### 问题3: Nginx 502 错误

**检查**:
```bash
sudo nginx -t
sudo systemctl status nginx
```

**解决**:
```bash
sudo systemctl restart nginx
```

### 问题4: 内存不足

**检查**:
```bash
free -h
pm2 monit
```

**解决**:
```bash
# 增加 swap
sudo fallocate -l 2G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
```
