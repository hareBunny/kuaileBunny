## 🔧 维护指南

### 添加更多测试数据

```bash
# 1. 生成SQL语句
cd backend-cloudflare
node add-demo-data.js

# 2. 执行SQL（复制生成的命令）
wrangler d1 execute kuaile8 --remote --command="INSERT INTO ..."
```

### 更新前端

```bash
# 1. 修改代码
cd h5
# 编辑文件...

# 2. 构建
npm run build

# 3. 部署
npx wrangler pages deploy dist --project-name=kuaile8 --commit-dirty=true
```

### 更新后端

```bash
# 1. 修改代码
cd backend-cloudflare
# 编辑文件...

# 2. 部署
wrangler deploy
```

### 查看日志

```bash
# 后端日志
wrangler tail kuaile8-backend

# 前端日志
# 在Cloudflare Dashboard → Pages → kuaile8 → Functions → Logs
```

## 📝 下一步工作

### 1. 配置真实数据爬虫

当前使用的是演示数据，需要配置真实API：

```bash
cd backend-cloudflare/crawler

# 方案1: 数字星空API（推荐）
# 1. 访问 https://szxk365.com 注册
# 2. 获取API KEY
# 3. 设置环境变量
set SZXK_API_KEY=your_key
# 4. 运行爬虫
npm run crawl:szxk

# 方案2: 极速数据API
# 1. 访问 https://www.jisuapi.com 注册
# 2. 获取API KEY
# 3. 设置环境变量
set JISU_APPKEY=your_key
# 4. 运行爬虫
npm run crawl:jisu
```

### 2. 配置定时任务

后端已配置每5分钟执行一次定时任务（Cron Trigger），但需要配置爬虫API才能自动获取数据。

### 3. 配置自定义域名（可选）

在Cloudflare Dashboard中：
1. Pages → kuaile8 → Custom domains
2. 添加你的域名
3. 配置DNS记录

### 4. 配置环境变量

后端需要配置以下密钥：

```bash
# JWT密钥
wrangler secret put JWT_SECRET

# Cron密钥
wrangler secret put CRON_SECRET

# 彩票API密钥（根据选择的API）
wrangler secret put LOTTERY_API_ID
wrangler secret put LOTTERY_API_KEY
```

## 🐛 问题排查

### 如果API返回"暂无数据"

说明数据库中没有数据，需要：
1. 运行 `node add-demo-data.js` 生成SQL
2. 使用 `wrangler d1 execute` 添加数据
3. 或配置真实爬虫自动获取数据

### 如果前端无法访问API

1. 检查 `h5/functions/_middleware.js` 中的后端地址
2. 确认后端已部署：`wrangler deploy`
3. 测试后端直接访问：`curl https://kuaile8-backend.2268043137.workers.dev/api/kuaile8/latest`

### 如果页面白屏

1. 按F12查看控制台错误
2. 检查Network标签，看哪个资源加载失败
3. 确认部署到的是Pages而不是Workers（URL应该是.pages.dev）
