# 快速部署指南 🚀

## 📋 部署流程总览

```
1. GitHub部署 → 2. Cloudflare Pages部署 → 3. Cloudflare Workers部署 → 4. 配置爬虫
```

---

## 🎯 第一步：部署到GitHub

### Windows用户

双击运行：
```
deploy-github.bat
```

### Mac/Linux用户

```bash
chmod +x deploy-github.sh
./deploy-github.sh
```

### 手动部署

```bash
# 1. 初始化Git
git init

# 2. 添加文件
git add .

# 3. 提交
git commit -m "feat: 快乐8百宝箱初始版本"

# 4. 设置主分支
git branch -M main

# 5. 添加远程仓库
git remote add origin https://github.com/hareBunny/kuaileBunny.git

# 6. 推送
git push -u origin main
```

### 常见问题

**问题1: 推送失败 - 认证失败**

解决方案：使用Personal Access Token

1. 访问 https://github.com/settings/tokens
2. 点击 **Generate new token (classic)**
3. 勾选 `repo` 权限
4. 生成并复制token
5. 推送时使用token作为密码

**问题2: 远程仓库不存在**

解决方案：先在GitHub创建仓库

1. 访问 https://github.com/new
2. 仓库名：`kuaileBunny`
3. 设为Private（推荐）
4. 不要初始化README
5. 创建后再推送

## 🌐 第二步：部署H5前端到Cloudflare Pages

### 方式A: 通过Dashboard（推荐新手）

1. **登录Cloudflare**
   - 访问 https://dash.cloudflare.com
   - 登录账号

2. **创建Pages项目**
   - 点击 **Workers & Pages**
   - 点击 **Create application**
   - 选择 **Pages** → **Connect to Git**

3. **连接GitHub**
   - 选择 **GitHub**
   - 授权Cloudflare
   - 选择 `hareBunny/kuaileBunny` 仓库

4. **配置构建**
   ```
   项目名称: kuaile8
   生产分支: main
   构建命令: cd h5 && npm install && npm run build
   输出目录: h5/dist
   ```

5. **添加环境变量**
   ```
   NODE_VERSION = 18
   VITE_API_BASE_URL = https://your-worker.workers.dev
   ```

6. **部署**
   - 点击 **Save and Deploy**
   - 等待2-3分钟
   - 获得访问地址：`https://kuaile8.pages.dev`

### 方式B: 使用CLI（推荐开发者）

```bash
# 1. 安装Wrangler
npm install -g wrangler

# 2. 登录
wrangler login

# 3. 构建前端
cd h5
npm install
npm run build

# 4. 部署
npx wrangler pages deploy dist --project-name=kuaile8

# 5. 获得访问地址
# https://kuaile8.pages.dev
```

## ⚡ 第三步：部署后端到Cloudflare Workers

### 1. 登录Cloudflare

```bash
cd backend-cloudflare
wrangler login
```

### 2. 创建D1数据库

```bash
# 创建数据库
wrangler d1 create kuaile8

# 记录返回的database_id，更新到wrangler.toml
```

### 3. 创建KV命名空间

```bash
# 创建KV
wrangler kv:namespace create CACHE

# 记录返回的id，更新到wrangler.toml
```

### 4. 初始化数据库

```bash
# 本地测试
wrangler d1 execute kuaile8 --file=./schema.sql --local

# 远程部署
wrangler d1 execute kuaile8 --file=./schema.sql --remote
```

### 5. 配置密钥

```bash
# JWT密钥（随机生成32位字符串）
wrangler secret put JWT_SECRET

# Cron密钥（随机生成32位字符串）
wrangler secret put CRON_SECRET

# API密钥（如果使用API爬虫）
wrangler secret put LOTTERY_API_ID
wrangler secret put LOTTERY_API_KEY
```

### 6. 部署Workers

```bash
wrangler deploy
```

### 7. 获取Workers地址

部署成功后会显示：
```
https://kuaile8-backend.your-account.workers.dev
```

### 8. 更新前端环境变量

回到Cloudflare Pages设置：
1. 进入 kuaile8 项目
2. 点击 **Settings** → **Environment variables**
3. 更新 `VITE_API_BASE_URL` 为Workers地址
4. 重新部署前端

## 🤖 第四步：配置数据爬虫

### 方式A: 使用免费API（推荐）

#### 1. 注册API KEY

访问 https://szxk365.com 注册获取API KEY

#### 2. 配置环境变量

```bash
# Windows
set SZXK_API_KEY=your_api_key

# Mac/Linux
export SZXK_API_KEY=your_api_key
```

#### 3. 测试爬虫

```bash
cd backend-cloudflare/crawler
npm install
npm run test:api
npm run crawl:szxk
```

#### 4. 启动定时任务

```bash
# 使用PM2
npm run pm2:start

# 查看状态
pm2 status

# 查看日志
pm2 logs kuaile8-crawler
```

### 方式B: 使用演示数据（测试用）

```bash
cd backend-cloudflare/crawler
npm run crawl:demo
```

## 🎨 配置自定义域名（可选）

### 1. 添加域名到Cloudflare

1. 登录Cloudflare Dashboard
2. 点击 **Add a site**
3. 输入你的域名
4. 选择免费计划
5. 更新域名DNS服务器

### 2. 配置Pages域名

1. 进入Pages项目
2. 点击 **Custom domains**
3. 点击 **Set up a custom domain**
4. 输入：`kuaile8.yourdomain.com`
5. 添加CNAME记录
6. 等待SSL证书配置

### 3. 配置Workers域名

1. 进入Workers项目
2. 点击 **Triggers** → **Custom Domains**
3. 添加：`api.yourdomain.com`
4. 配置DNS记录

## 🐛 故障排查

### 前端问题

**页面空白**
- 检查浏览器控制台错误
- 检查API地址配置
- 检查路由配置

**API请求失败**
- 检查CORS配置
- 检查Workers是否部署
- 检查环境变量

### 后端问题

**Workers部署失败**
- 检查wrangler.toml配置
- 检查TypeScript错误
- 查看部署日志

**数据库连接失败**
- 检查database_id
- 检查数据库是否创建
- 重新初始化数据库

### 爬虫问题

**爬虫不运行**
- 检查PM2状态
- 检查API KEY配置
- 查看错误日志

**数据不更新**
- 检查定时任务配置
- 手动执行测试
- 检查API限制
