# 本地开发启动指南

## 方案一：使用 Cloudflare Workers 本地开发

### 1. 启动后端（Cloudflare Workers）

```bash
cd backend-cloudflare
npm run dev
```

后端运行在：http://localhost:8787

### 2. 启动 H5 前端

打开新终端：

```bash
cd h5
npm run dev
```

前端运行在：http://localhost:5173

### 3. 测试

访问 http://localhost:5173 即可看到完整应用。

前端会自动代理 API 请求到后端。

---

## 方案二：使用 Next.js 后端（如果不想用 Cloudflare）

### 1. 配置 MongoDB

编辑 `backend/.env.local`：

```bash
# 使用 MongoDB Atlas（推荐）
MONGODB_URI=mongodb+srv://username:password@cluster.mongodb.net/kuaile8

# 或使用本地 MongoDB
MONGODB_URI=mongodb://localhost:27017/kuaile8
```

### 2. 启动后端

```bash
cd backend
npm install
npm run dev
```

后端运行在：http://localhost:3001

### 3. 修改 H5 前端配置

编辑 `h5/vite.config.ts`：

```typescript
proxy: {
  '/api': {
    target: 'http://localhost:3001',  // 改为 Next.js 端口
    changeOrigin: true
  }
}
```

### 4. 启动 H5 前端

```bash
cd h5
npm run dev
```

前端运行在：http://localhost:5173

## 生产环境部署

### 选项 1: 使用自定义域名部署到 Cloudflare

如果你有自己的域名：

1. 在 Cloudflare 添加域名
2. 部署 Worker：
   ```bash
   cd backend-cloudflare
   npm run deploy
   ```
3. 在 Cloudflare Dashboard 配置自定义域名
4. 更新 H5 的 API 地址

### 选项 2: 部署到 Vercel

#### 后端（Next.js）

```bash
cd backend
vercel
```

#### 前端（H5）

```bash
cd h5
npm run build
vercel
```

### 选项 3: 部署到服务器

#### 后端

```bash
cd backend
npm run build
pm2 start npm --name "kuaile8-api" -- start
```

#### 前端

```bash
cd h5
npm run build
# 将 dist 目录部署到 Nginx
```

## 常见问题

### Q: Cloudflare Workers 本地开发报错？

A: 确保已安装依赖：
```bash
cd backend-cloudflare
npm install
```

### Q: 数据库没有数据？

A: 初始化数据库：
```bash
# Cloudflare D1
cd backend-cloudflare
wrangler d1 execute kuaile8 --file=./schema.sql

# MongoDB
cd backend
npm run db:init
npm run db:seed
```

### Q: 前端无法连接后端？

A: 检查：
1. 后端是否正常运行
2. 端口是否正确
3. vite.config.ts 中的 proxy 配置是否正确

### Q: 如何切换后端？

A: 修改 `h5/vite.config.ts` 中的 proxy target：
- Cloudflare Workers: `http://localhost:8787`
- Next.js: `http://localhost:3001`

## 推荐开发流程

1. **本地开发**：使用 Cloudflare Workers 本地开发（无需数据库配置）
2. **测试功能**：在本地完整测试所有功能
3. **准备部署**：
   - 有域名：部署到 Cloudflare + 自定义域名
   - 无域名：部署到 Vercel（Next.js 版本）
4. **上线运营**：配置支付接口、监控等
