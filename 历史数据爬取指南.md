# 快乐8历史数据爬取指南

## 📋 概述

本指南说明如何批量爬取快乐8的历史数据，用于初始化数据库。

---

## 🚀 快速开始

### 方法1: 使用批处理文件（推荐）

**测试爬取（7天）**:
```bash
crawl-history-test.bat
```

**完整爬取（一年）**:
```bash
crawl-history-year.bat
```

```bash
cd backend-cloudflare/crawler

# 爬取最近7天
node mxnzp-history.js 7

# 爬取最近30天
node mxnzp-history.js 30

# 爬取最近90天
node mxnzp-history.js 90

# 爬取最近一年
node mxnzp-history.js 365
```

### 方法3: 使用npm脚本

```bash
cd backend-cloudflare/crawler

# 爬取一年（默认）
npm run crawl:mxnzp:history

# 爬取30天
npm run crawl:mxnzp:history:30

# 爬取90天
npm run crawl:mxnzp:history:90

# 爬取365天
npm run crawl:mxnzp:history:365
```

## 📊 数据说明

### 数据来源

- API: MXNZP (https://www.mxnzp.com)
- 接口: `/api/lottery/common/history`
- 彩种: 快乐8 (kl8)

### 数据格式

每天约50期开奖数据，每期包含：
- 期号 (expect)
- 开奖号码 (20个号码)
- 开奖时间
- 号码总和

### 数据存储

数据保存到Cloudflare D1数据库：
- 表名: `lottery_results`
- 去重: 使用期号作为唯一键
- 冲突处理: 更新已存在的数据

## ⚠️ 注意事项

### API限制

- **QPS限制**: 普通会员1次/秒
- **请求间隔**: 脚本已设置1.5秒间隔
- **超限处理**: 如果请求过快会返回错误

### 网络问题

如果遇到网络错误：
1. 检查网络连接
2. 等待几分钟后重试
3. 减少爬取天数
4. 分批次爬取

### 数据完整性

- 脚本会自动跳过无数据的日期
- 网络错误不会中断整个流程
- 最后会显示成功/失败统计

## 🛠️ 故障排查

### 问题1: 请求频率过快

**错误信息**: `请求频率过快，超过当前账号QPS的限制`

**解决方法**:
- 增加请求间隔（修改 `DELAY` 配置）
- 等待一段时间后重试
- 升级API会员等级

### 问题2: 网络连接失败

**错误信息**: `fetch failed` 或 `网络错误`

**解决方法**:
- 检查网络连接
- 检查防火墙设置
- 尝试使用VPN
- 稍后重试

### 问题3: 数据库保存失败

**错误信息**: `保存到数据库失败`

**解决方法**:
- 确认wrangler已登录: `wrangler whoami`
- 检查数据库连接
- 查看错误日志

### 问题4: 部分数据失败

**现象**: 显示 "保存 49/50 条"

**说明**: 
- 这是正常的，可能是网络波动
- 失败的数据会在日志中显示
- 可以重新运行脚本补充数据（会自动去重）

## 💡 优化建议

### 分批爬取

如果一次性爬取一年太慢，可以分批：

```bash
# 第一批: 最近3个月
node mxnzp-history.js 90

# 第二批: 3-6个月前
# 需要修改脚本支持日期范围

# 第三批: 6-12个月前
# 需要修改脚本支持日期范围
```

### 并行爬取

**不推荐**: 会超过QPS限制

### 定时补充

创建定时任务，每周补充一次历史数据：

```bash
# 每周日凌晨补充最近7天
node mxnzp-history.js 7
```

## 📊 数据统计示例

爬取完成后，可以进行各种统计分析：

### 号码频率统计

```sql
SELECT 
  number,
  COUNT(*) as frequency
FROM (
  SELECT json_each.value as number
  FROM lottery_results, json_each(numbers)
  WHERE lottery_type = 'kuaile8'
)
GROUP BY number
ORDER BY frequency DESC
LIMIT 10
```

### 按月统计

```sql
SELECT 
  strftime('%Y-%m', draw_date/1000, 'unixepoch') as month,
  COUNT(*) as count
FROM lottery_results
WHERE lottery_type = 'kuaile8'
GROUP BY month
ORDER BY month DESC
```
