# Cloudflare Pages 部署问题排查清单

## 📋 快速诊断

按顺序检查以下项目，找到问题所在。

---

## ✅ 第一步: 验证本地构建

### 运行验证脚本

```bash
verify-build.bat
```

### 手动检查

```bash
cd h5
npm run build
```

**期望结果**:
```
✓ built in 2.5s
dist/index.html
dist/assets/index-xxx.js
dist/assets/index-xxx.css
```

**检查文件**:
- [ ] `h5/dist/index.html` 存在
- [ ] `h5/dist/assets/` 目录存在
- [ ] `h5/dist/assets/` 包含 .js 和 .css 文件
- [ ] `h5/dist/_redirects` 存在
- [ ] `h5/dist/_headers` 存在

**查看index.html**:
```bash
type h5\dist\index.html
```

应该看到：
```html
<script type="module" crossorigin src="/assets/index-xxx.js"></script>
```

✅ 如果路径是 `/assets/xxx.js` → 构建正确
❌ 如果路径是 `/src/main.tsx` → 构建错误

---

## ✅ 第二步: 检查Cloudflare Dashboard配置

### 登录Dashboard

访问: https://dash.cloudflare.com

### 导航到项目

Workers & Pages → kuaile8

### 检查配置

Settings → Builds & deployments → Build configuration

**必须的配置**:

| 配置项 | 正确值 | 常见错误 |
|--------|--------|----------|
| Framework preset | None | - |
| Build command | `cd h5 && npm install && npm run build` | 缺少 `cd h5` |
| Build output directory | `h5/dist` | `dist`, `/dist`, `./h5/dist` |
| Root directory | (留空) | 填了 `h5` |
| Node version | 18 | 未设置 |

**检查清单**:
- [ ] Build command 包含 `cd h5`
- [ ] Build output directory 精确是 `h5/dist`（无前缀、无后缀）
- [ ] Root directory 是空的
- [ ] Environment variables 包含 `NODE_VERSION=18`

### 如何修改

1. 点击 **Edit configuration**
2. 逐项修改
3. 点击 **Save**
4. 回到 Deployments 标签
5. 点击最新部署 → **Retry deployment**
6. 勾选 **Clear build cache**

---

## ✅ 第三步: 检查构建日志

### 查看日志

Deployments → 点击最新部署 → View build log

### 关键检查点

#### 1. 构建命令执行

应该看到：
```
Executing user build command: cd h5 && npm install && npm run build
```

✅ 如果看到 `cd h5` → 命令正确
❌ 如果只看到 `npm run build` → 命令错误

#### 2. 构建成功

应该看到：
```
✓ built in 2.5s
dist/index.html
dist/assets/index-xxx.js
```

✅ 如果看到 `dist/` 开头的文件 → 构建成功
❌ 如果看到错误信息 → 构建失败

#### 3. 文件上传

应该看到：
```
Uploading...
✨ Success! Uploaded 8 files
  ✓ index.html
  ✓ assets/index-xxx.js
  ✓ assets/index-xxx.css
  ✓ _redirects
  ✓ _headers
```

✅ 如果上传的是 `assets/` 文件 → 正确
❌ 如果上传的是 `src/` 文件 → 配置错误

#### 4. 部署成功

应该看到：
```
✨ Deployment complete!
https://xxx.kuaile8.pages.dev
```

## ✅ 第四步: 测试部署的网站

### 访问网站

https://kuaile8.pages.dev

### 检查浏览器控制台

按 F12 打开开发者工具

#### Console 标签

**正常情况**: 无错误或只有警告

**异常情况**: 
```
main.tsx:1 Failed to load module script: Expected a JavaScript module script 
but the server responded with a MIME type of "".
```

如果看到这个错误：
- 错误指向 `main.tsx` → Cloudflare在加载源文件
- 说明 Build output directory 配置错误
- 立即检查第二步的配置

#### Network 标签

刷新页面，查看请求：

**正常情况**:
```
index.html          200  text/html
index-xxx.js        200  application/javascript
index-xxx.css       200  text/css
```

**异常情况**:
```
main.tsx            404  -
```

如果请求的是 `main.tsx`：
- 说明 index.html 中引用了源文件
- 说明部署的是源代码，不是构建文件
- Build output directory 配置错误

### 直接访问资源文件

尝试访问：
```
https://kuaile8.pages.dev/assets/index-xxx.js
```

（从Network标签复制实际的文件名）

**正常情况**: 返回压缩后的JS代码
**异常情况**: 返回404或源代码

## ✅ 第五步: 清除缓存

### 浏览器缓存

按 Ctrl + Shift + R 强制刷新

或者：
1. F12 打开开发者工具
2. 右键点击刷新按钮
3. 选择"清空缓存并硬性重新加载"

### Cloudflare缓存

1. Cloudflare Dashboard → kuaile8
2. Settings → Functions
3. 点击 **Purge build cache**
4. 回到 Deployments
5. Retry deployment

## 🎯 最常见的3个问题

### 问题1: Build output directory 配置错误（90%的情况）

**症状**: 浏览器显示 `main.tsx:1 Failed to load module script`

**解决**: 
1. Dashboard → Settings → Builds & deployments
2. 修改 Build output directory 为 `h5/dist`
3. Retry deployment

### 问题2: 浏览器缓存（5%的情况）

**症状**: 配置正确但仍然白屏

**解决**: Ctrl + Shift + R 强制刷新

### 问题3: Cloudflare缓存（5%的情况）

**症状**: 重新部署后仍然显示旧版本

**解决**: 
1. Settings → Functions → Purge build cache
2. Retry deployment
